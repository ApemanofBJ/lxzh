由 Apache 开发的一个 Servlet 容器(.jsp-->.java 的编译)，实现了对 Servlet 和 JSP (Java Server Page，#嵌入HTML，框架SSH--Structs,Spring, Hebernate,#Jasper编译为Servlet)的支持
特点：运行时占用的系统资源小，扩展性好，支持负载平衡与邮件服务等开发应用系统常用的功能

    架构:
  • Server -> Connector
  • Service 将连接器Connector与引擎进行关联
  • Engine --Servlet Container,定义时需要指定默认主机
  • Host 在引擎内部实现虚拟主机
  • context 网站文件路径
    组件工具
  • realm:用户账号数据库
  • valve(过滤器):记录访问日志、基于ip认证
  • logger:日志记录器,不能用于context
    部署方式
  • 单独服务器方式响应WEB需求
  • 代理服务器方式:
    apache+tomcat--使用专用代理服务器模块与后端集群进行连接，协议ajp(apache jserv protocol)效率高
    nginx+tomcat--协议http，使用upstream

[安装]

wget http://mirrors.hust.edu.cn/apache/tomcat/tomcat-8/v8.5.24/bin/apache-tomcat-8.5.24.tar.gz

配置文件-[server.xml]
port #用telnet通过端口访问 a.> 端口
Connector port="8080"

catalina.policy 网络访问的安全策略 webapps 应用程序目录(网页) work 工作目录

启动程序 -建立catalina.sh
export CATALINA_HOME=/usr/local/tomcat
export PATH=$PAHTH:$CATALINA_HOME/bin
exec $CATALINA_HOME/bin/catalina.sh $*
